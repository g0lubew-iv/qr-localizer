Тестирование разработанного алгоритма локализации выполнялось на наборе изображений с ручной разметкой (Ground Truth), подготовленной в инструменте LabelMe.
Для каждого изображения алгоритм либо возвращал четырёхугольник области QR-кода (четыре вершины в порядке \texttt{TL-TR-BR-BL}), либо сообщал об отсутствии результата.
Далее результат сравнивался с Ground Truth по метрикам IoU и ошибкам по углам.

Для оценки точности локализации использовались следующие метрики:

\begin{itemize}
	\item \textbf{IoU (Intersection over Union)} --- отношение площади пересечения области, найденной алгоритмом, к площади объединения найденной области и эталонной разметки.
	      Значение $IoU \in [0, 1]$, где 1 соответствует полному совпадению.
	\item \textbf{Средняя ошибка по углам} (\texttt{corner\_err\_mean\_px}) --- среднее евклидово расстояние (в пикселях) между соответствующими вершинами предсказанного многоугольника и разметки:
	      \[
		      e_{mean}=\frac{1}{4}\sum_{i=1}^{4} \|q_i - q_i^{GT}\|.
	      \]
	\item \textbf{Максимальная ошибка по углам} (\texttt{corner\_err\_max\_px}) --- максимальная ошибка среди четырёх вершин:
	      \[
		      e_{max}=\max_{i\in\{1,2,3,4\}} \|q_i - q_i^{GT}\|.
	      \]
\end{itemize}

Метрики по углам позволяют дополнительно оценить характер ошибки (например, наличие одного сильно <<съехавшего>> угла), даже если IoU остаётся относительно высоким.
Результаты тестирования доступны в файле \texttt{data/results.csv}.

\subsubsection{Сводные результаты}
По приведённой выборке ($35$ изображений, на всех алгоритм вернул результат) получены следующие значения:
\begin{itemize}
	\item \textbf{Средний IoU:} $\approx 0.966$.
	\item \textbf{Медианный IoU:} $\approx 0.974$.
	\item \textbf{Минимальный IoU:} $0.9185$ (файл \texttt{test29.jpg}).
	\item \textbf{Максимальный IoU:} $0.9841$ (файл \texttt{test0.jpg}).
\end{itemize}

\subsubsection{Анализ по группам качества}
Для более содержательного анализа результаты были разделены на группы по значению IoU.

\paragraph{Высокое качество (IoU $\ge 0.98$).}
К этой группе относятся 6 изображений: \texttt{test0.jpg} (IoU 0.9841),
\texttt{test11.jpg} (0.9806), \texttt{test17.jpg} (0.9839), \texttt{test18.jpg} (0.9819), \texttt{test28.jpg} (0.9814),  \texttt{test12.jpg} (0.9787).

Для этих случаев средняя ошибка углов не превышает 5 пикселей
(диапазон 1.6-4.5 px), а максимальная ошибка составляет 5-8 пикселей.
Это соответствует практически идеальному совпадению предсказанных и истинных
границ QR-кода с точностью до нескольких пикселей.

	\caption{Файл \texttt{test0.jpg}: высокое качество (IoU 0.9841)}
	\label{fig:a4}
\end{figure}

\paragraph{Хорошее качество ($0.90 \le$ IoU $< 0.95$).}
Наиболее многочисленная группа -- 20 изображений: \texttt{test2.jpg} (0.9760),
\texttt{test6.jpg} (0.9761), \texttt{test8.jpg} (0.9754),
\texttt{test9.jpg} (0.9739), \texttt{test10.jpg} (0.9755),
\texttt{test13.jpg} (0.9748), \texttt{test14.jpg} (0.9766),
\texttt{test15.jpg} (0.9768), \texttt{test19.jpg} (0.9778),
\texttt{test24.jpg} (0.9754), \texttt{test25.jpg} (0.9692),
\texttt{test7.jpg} (0.9660), \texttt{test20.jpg} (0.9710),
\texttt{test21.jpg} (0.9700).

Средняя ошибка углов в этой группе составляет 4-7 пикселей, максимальная ошибка достигает 8-14 пикселей.
Характерно небольшое ухудшение точности позиционирования отдельных углов,
что связано с незначительными перспективными искажениями.

\paragraph{Удовлетворительное качество ($0.91 \le$ IoU $< 0.97$).}
К этой группе относятся 7 изображений: \texttt{test16.jpg} (0.9724),
\texttt{test22.jpg} (0.9197), \texttt{test24.jpg} (0.9619),
\texttt{test26.jpg} (0.9521), \texttt{test29.jpg} (0.9185),
\texttt{test30.jpg} (0.9557), \texttt{test31.jpg} (0.9420),
\texttt{test32.jpg} (0.9369).

Здесь наблюдается заметное увеличение ошибок: средняя ошибка углов достигает 5-11 пикселей,
а максимальная может превышать 15 пикселей (до 15.8 px на \texttt{test22.jpg} и 15.7 px на \texttt{test32.jpg}).

Основные причины снижения точности в этой группе:
\begin{itemize}
	\item Сложные условия съёмки (сильная перспектива, наклон камеры);
	\item Нестандартное соотношение сторон изображений (например, \texttt{test22.jpg} имеет разрешение 4344×4578, а \texttt{test29.jpg} -- 5786×2185).
\end{itemize}

Например, в случае с тестовым изображением \texttt{test22.jpg}, имеем следующую причину низкой точности локализации:
неточное определение центра Finder Pattern'ов, в связи с чем произошёл сдвиг влево правой границы QR-кода.
